Hello, named pipe

发表于 2017-09-28 | 更新于 2017-10-07 |

初识 named pipe

Pipe 是我们在使用 linux 时经常会使用到的的一个工具，它可以帮助我们把两个独立的程序组合在一起。如命令 ls | grep hi 可以帮助我们找出当前路径下所有文件名中包含 ‘hi’ 的文件。

在这篇文章中我们要介绍的是另一个概念：named pipe, 它和 pipe 都属于常见的进程间通信 方式。 Named pipe 也被称作 FIFO(First In, First Out)。Named pipe 中的 ‘name’ 实际上指的是文件名。在ls -l命令中，pipe 文件的 mode string 1 列会被标注为 p。

Named pipe 的简单使用

在较新的 linux 系统上，named pipe 可以用 mkfifo 命令创建（在早先的 linux 版本中则为 mknod 命令）。

首先，用mkfifo example_p指令创建一个名叫 ‘example_p’的 named pipe；然后在一个窗口中运行命令ls -l > example_p，在另一个窗口中运行命令cat <example_p。我们将会看到，ls -l的结果输出在了第二个窗口中。

一个由 named pipe 组成的循环

在一个窗口中输入指令echo -n x | cat - pipe1 > pipe2 &，在另一个窗口中输入指令cat <pipe2 > pipe1。虽然屏幕上没有任何输出，但当你用 top 命令查看当前的系统进程，就会发现 cat 命令在疯狂地运行 (cat 在将 x 来来回回地复制）。

如果用 Ctrl+C 强制终止第二行指令，可能会出现提示”broken pipe”（也可能不会出现，原因见下一段）。这个错误提示表示一个进程将数据写入 pipe 的时候，pipe 另一端的进程终止了，数据无处可去了。通常情况下，写入者这时会停止写入数据并关闭 pipe。

错误提示”broken pipe”是否会出现取决于用户执行 Ctrl+C 的时机。如果用户执行 Ctrl+C 的时候第二个 cat 刚好写入了数据（即 x 刚好写入了 pipe1），那么 pipe1 会被关闭，且第一个 cat 会因此立即终止。这种情况下就不会出现任何提示。

而在另一种情况下，第二个 cat 正在等待第一个 cat 将 x 写入 pipe2，而用户执行了 Ctrl+C 使其终止，也使 pipe2 关闭，导致第一个 cat 无法将数据写入 pipe2。这时就会出现错误提示”broken pipe”。
